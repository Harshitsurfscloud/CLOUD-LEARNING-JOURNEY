4. Git init - (It creates a hidden folder name {.git} which tracks all

   -Branch information

5. Git status - shows the status of the file whether tracked or untracked

6. Git add - add the the file to the staging area.

7. Git commit - after conmfirmation it takes the snapshot of the file and saves it.

8. Git checkout  - it takes back to the previous commit if some changes are made mistakenly.

9. Git log - gives detail for the user and about the previous commit.

10. Git diff - tell weather there in any diffrences made in the file.
